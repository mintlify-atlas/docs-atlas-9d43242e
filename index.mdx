Seguridad App is a backend security platform built with Ktor and Kotlin. It provides JWT authentication, granular role-based access control, user and profile management, and a module/menu hierarchy — all exposed through a clean REST API.

## What Seguridad App does

Seguridad App gives you a complete security layer you can integrate into any web application:

- **JWT Authentication** — Users log in and receive a signed token valid for 24 hours. All protected endpoints require this token as a Bearer header.
- **Role-based profiles** — Every user belongs to a profile. Profiles marked as administrator have unrestricted access; others are controlled by per-module permissions.
- **Granular permissions** — Each profile can be granted add, edit, view, delete, and detail rights on a per-module basis.
- **User management** — Create, update, and deactivate users. Upload profile photos via multipart form.
- **Module & menu hierarchy** — Modules represent features in your application. Menus group modules for navigation.
- **reCAPTCHA protection** — Login requires a valid Google reCAPTCHA v2 token to prevent automated attacks.

## Getting started

<Steps>
  <Step title="Clone and configure">
    Clone the repository and set your environment variables — database URL, JWT secret, and reCAPTCHA keys.
  </Step>
  <Step title="Start the server">
    Run `./gradlew run` or use Docker. The server starts on port 8080 and creates all database tables automatically.
  </Step>
  <Step title="Log in">
    Send a `POST /api/auth/login` request with credentials and a reCAPTCHA token. You'll receive a JWT to use in subsequent requests.
  </Step>
  <Step title="Call protected endpoints">
    Pass the token as `Authorization: Bearer <token>` on any resource endpoint to manage users, profiles, modules, and permissions.
  </Step>
</Steps>

<Note>
  The server seeds an `admin` / `Admin123` user on first start. Change this password immediately in a production deployment.
</Note>
