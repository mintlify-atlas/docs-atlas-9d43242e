Seguridad App is a backend security service built with Ktor and Kotlin. It handles the full authentication and authorization lifecycle for web applications: users log in, receive a signed JWT, and every subsequent request is validated against their profile's granular per-module permissions. The server manages users, roles (profiles), modules, menus, and permissions through a clean REST API backed by PostgreSQL.

## Key features

**Authentication**
- JWT authentication using HMAC-256 signing via JJWT 0.12.5
- Tokens are valid for 24 hours (`86400000 ms`), carrying `userId`, `perfilId`, and `esAdministrador` claims
- Google reCAPTCHA v2 validation on every login request to block automated attacks
- bcrypt password hashing with cost factor 12 via `at.favre.lib:bcrypt`

**Authorization**
- Role-based access control via profiles (`Perfiles`) â€” every user belongs to exactly one profile
- Profiles flagged as administrator receive all permissions automatically; all other profiles are controlled by explicit per-module grants
- Granular permissions per module: add (`bitAgregar`), edit (`bitEditar`), view (`bitConsulta`), delete (`bitEliminar`), and detail (`bitDetalle`)

**User and profile management**
- Full CRUD for users, including active/inactive status control
- Profile photo upload via multipart form
- Initial admin user and two default profiles seeded automatically on first start

**Module and menu hierarchy**
- Modules represent discrete features or sections of your application
- Menus group modules for navigation â€” a `MenuModulos` join table links them
- Eight seed modules and three seed menus created on first start

**Infrastructure**
- PostgreSQL database with Exposed ORM and HikariCP connection pooling (max 10 connections, `REPEATABLE_READ` isolation)
- SSL required on all database connections
- Multi-stage Docker build â€” final image is `eclipse-temurin:21-jre-jammy`
- Fat JAR (`seguridad-app-all.jar`) for standalone deployment
